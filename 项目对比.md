## 各 AI 科研项目沙箱/容器适配及迁移参考表
### 优先参考项目

| 项目             | 沙箱 / 容器情况说明                                          | 需优先迁移的目录 / 文件                                      |
| ---------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| AutoResearchClaw | 内置`experiment.mode`支持 sandbox/docker/ssh_remote；含加固 Docker 沙箱、网络策略、内存 / GPU 资源限制等完整配置 | researchclaw/、config.researchclaw.example.yaml、tests/、docs/ |
| DeerFlow         | 具备`SandboxProvider`抽象层，原生支持本地沙箱、Docker 隔离 Aio 沙箱两种实现 | backend/packages/harness/deerflow/sandbox/、backend/packages/harness/deerflow/community/、backend/docs/CONFIGURATION.md、backend/docs/ARCHITECTURE.md、docker/、config.example.yaml |
| AI-Researcher    | 基于 Docker 搭建 Agent 交互式运行环境，支持工作目录、基础镜像、容器名、GPU、端口等完整配置 | docker/、research_agent/、.env.template、web_ai_researcher.py |
| RD-Agent         | 环境执行层成熟完善，核心`rdagent/utils/env.py`内置 Docker 配置、超时控制、数据卷、GPU / 内存 / CPU 限制、缓存机制 | rdagent / 全目录、.env.example、docs/                        |
| AI-Scientist v1  | 支持容器化，以运行环境 Dockerfile 为主，无完整沙箱管理框架   | experimental/Dockerfile、ai_scientist/、templates/、launch_scientist.py |
| AI-Scientist v2  | 文档明确要求 Docker 受控沙箱运行 LLM 生成代码，仓库无内置完整沙箱管理层 | ai_scientist/、bfts_config.yaml、launch_scientist_bfts.py    |

### 次优先 / 可选参考项目

| ---------------------------- | ------------------------------------------------------------ |
| Auto-Deep-Research           | 支持 Docker 容器化 Agent 交互环境；代码沙箱 / E2B 仍在 Todo 阶段，仅基础容器能力，无成熟沙箱管理；可迁移：autoagent/、loop_utils/、.env.template |

| OpenHands / SWE-agent / AIDE | 代码执行基础设施级项目，沙箱底层实现强悍；非纯自动科研场景，适合后期工程化沙箱单独参考 |

| ARIS / claude-scholar | 依赖 Claude Code 等外部代码执行器，无自研内置平台化沙箱      |
